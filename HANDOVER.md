## 実装済み機能（v5時点）

### キャンバス・描画
- ペン 1px / 2px / 20px、アンチエイリアスOFF
- インクカラー 3色（主線黒・下書き水色・指示オレンジ）
- 消しゴム（ペンと独立してサイズ記憶）
- 塗りつぶし（スキャンラインフラッドフィル）
- 48色パレット（右サイドパネル）
- ストローク補正（4点移動平均）
- 背景明度縦スライダー（左ツールバー）
- ピンチズーム＋パン（キャンバス）
- Undo/Redo（履歴50ステップ、フレーム単位独立）

### ツールバー構成
- **左**: ペン・消し・塗り / サイズ1px・2px・20px / 主線・下書・指示 / BG縦スライダー
- **右**: 48色FILL PALETTEのみ（他は全部左に移動済み）
- UNDO/REDOボタンは削除（キーボードとジェスチャーで代替）

### タイムライン
- `kind: 'draw' | 'empty'` の2種セルで構成
  - `draw`: 描画フレーム（サムネ表示）
  - `empty`: 空ブロック（斜線パターン、タップで draw に昇格）
- duration（コマ打ち数）:
  - 1〜3コマ: 固定幅 + 内側仕切り線
  - 4コマ以上: セル横伸縮 + 下部目盛り
  - セル右端スクラブで 1〜120コマまで自由変更（最大15秒制限）
- 末尾エリアタッチ: 「前フレームを伸ばす + 1コマの新フレーム追加」
- セル並び替え: ペン/マウスでドラッグ
- **指でタイムラインを横スワイプ = ネイティブスクロール**
- Alt+ホイールでセル基準幅伸縮

### ワークエリア（時間メモリ上部）
- ネオンイエローの帯で再生・書き出し範囲を表示
- 左右ハンドルドラッグで範囲指定
- 中央ボディドラッグで範囲移動
- ダブルタップで「全範囲 → 現在フレームのみ → 元の状態」サイクル
- 再生はワークエリア内のみをループ
- 最大15秒制限

### 操作系
- **Apple Pencil**: 常に描画
- **指1本**: ペンツール時=パン、消しゴム/塗りつぶし時=描画
- **指2本**: ピンチズーム + パン
- **指2本タップ**: Undo
- **指3本タップ**: Redo
- **キーボード**: B=ペン、E=消し、G=塗り、1/2/3=サイズ、O=オニオン、N=新規、D=複製、Space=再生、←→=コマ移動、Del/Backspace=削除、Ctrl+Z/Y=Undo/Redo

### 保存・書き出し
- IndexedDB自動保存（800ms遅延、差分保存）
- 4K PNG書き出し（現在のフレームのみ、BG白固定）

## 次回実装予定（第3便B）

### 優先度高
1. **複数選択**
   - フレームセルをShift+クリック or タップで追加選択
   - 選択中セルはハイライト表示
2. **コピー/ペースト**
   - コピー: 選択フレームをクリップボード（JS変数）に
   - ペースト: currentFrameの直後に挿入
3. **複数選択での丸ごと移動**
   - 選択コマ以降全部をセットで移動
   - 元位置には `empty` ブロックが残る（長さ維持）
4. **複数削除**
   - 選択中のフレームをまとめて削除

### 優先度中
5. **4K連番PNG書き出し（zip）**
   - 全フレームをオフスクリーン4Kで再描画
   - JSZipでzip圧縮してダウンロード
   - BG透明オプション
6. **FILL PALETTEのフローティング化**
   - 右パネルから切り離してドラッグ移動できるウィンドウに
7. **参照画像パネル**
   - フローティングUI
   - 画像読み込み（File API）
   - ホイール拡縮 + パン
   - スポイトでFILL PALETTEに色取り込み

### 優先度低（将来）
8. **タイムライン共通レイヤー**
   - 全コマ共通の背景レイヤー（トレース用画像置き場）
   - 各コマ側にレイヤーは持たない（軽量化）
9. **PWA化**
   - manifest.json + service worker
   - GitHub Pages経由でSafari「ホーム画面に追加」
10. **GitHubエクスポート/インポート**
    - プロジェクトをzip（JSON+PNG）でエクスポート
    - インポートで復元

## 既知の問題・注意点

- IndexedDBの旧バージョン（v1）からv2への移行コードあり（frameOrderの形式変換）
- 再生時は描画の同期（syncFrameFromCanvas）を行わないため高速
- 空ブロックはDrawDataを持たない。オニオンスキンは空ブロックを飛ばして前後のdrawを参照
- 再生はワークエリア範囲内のみ。空ブロックの間は直前のdrawフレームを表示し続ける
- スクラブハンドルの幅は8px（タッチ用に ±8px のヒット領域拡張あり）
- `confirm()` / `alert()` の代わりにカスタムモーダル使用（iPad HTMLビューア対応）
- Apple Pencilの描き始めに若干の反応遅延あり（許容範囲とのこと、将来改善候補）
